Text Preprocessing¶

Text preprocessing is a method to clean and prepare text data for learning by the model. Text data contains noise in various forms such as emoticons, punctuation marks, and different capitalization. Raw text data may contain unwanted or unimportant text, so machine learning models may produce unsatisfactory accuracy. We should not input raw data without preprocessing to build a model because preprocessing text directly improves model performance. In some cases, if we input raw data without preprocessing techniques, the model will get confused and give random results.

Case Folding¶

Case folding is the process of uniformizing the text letters of all documents in the corpus. In text classification, case folding is generally done by homogenizing the document text in the corpus into lowercase. Lowercase folding is done to homogenize all letter shapes into lowercase letters so that there are no differences in words/terms at the time of feature extraction due to differences in letter shapes.

Text Cleansing¶

Text cleansing consists of removing unnecessary data attributes such as certain punctuation marks (periods, commas, semicolons, quotes, etc.) and Uniform Resource Locator (URL) addresses. Specifically for Twitter data, some other attributes are not needed in the classification process, namely the RT (re-tweet) character and username (the name of the account that wrote the tweet). URLs, RT/mentions, and product names in this dataset have been replaced with tags.

During text cleaning, it is common to use regular expression (regex). A regex is a sequence of characters that define search patterns in text. Here's a reference to learn more about regex:

  • https://www.w3schools.com/python/python_regex.asp
  • https://regex101.com/

Regular Expression¶

Regular expression or regex can be used to capture patterns in text. This Tool is very useful when we have some patterns that we want to replace with certain characters.

Suppose we want to replace all numbers with spaces. Numbers have a range from 0-9, so that we don't have to replace the numbers one by one, we can use regex.

Regular expression in Python can be done with the re library.

Patterns¶

The parentheses [ ] hold the patterns we want to search for

Suppose we want to replace the question mark ? and exclamation mark ! with a full stop .

We can also replace this with empty string

\w and \W match any word character

\w matches any alphanumeric word character. Word characters are uppercase and lowercase letters, numbers, and the underscore character ( _ ):

'      -  -*/'

\W is the opposite. \W matches any character other than alphanumeric and underscore:

\d and \D match numbers (digits).

\d matches any number character. Conversely, \D matches any character that is not a number.

Quantifiers¶

A Quantifier indicates how many times a part must appear for a successful match.

Plus (+) matches one or more repetitions.

In [ ]:
print(re.sub('\s', '/', 'masa setiap hari  pulsa berkurang 1500-2100 bisa 2-3*/hari'))   # tanpa +

print(re.sub('\s+', '/', 'masa setiap hari  pulsa berkurang 1500-2100 bisa 2-3*/hari'))  # dengan +

masa/setiap/hari//pulsa/berkurang/1500-2100/bisa/2-3*/hari
masa/setiap/hari/pulsa/berkurang/1500-2100/bisa/2-3*/hari

The plus sign (+) captures the repetition of the pattern more than 1x, so in the example above, the double space is only replaced by 1 slash (/). This step will be very useful when we want to reduce the repeated pattern to just 1 pattern.

Example: we want to replace double spaces with 1 space

def text_cleansing(text):
    
    # remove tags
    text = text.replace('<provider_name>', '')
    text = text.replace('<url>', '')
    text = text.replace('<user_mention>', '')
    text = text.replace('<product_name>', '')
    
    # remove non ASCII (emoticon, chinese word, etc)
    text = text.encode('ascii', 'replace').decode('ascii')
    
    # remove digits (using regex) -> subtitute
    text = re.sub('\d', '', text)
    
    # remove punctuation
    text = re.sub('[!"#$%&\'()*+,-./:;<=>?@[\\]^_`{|}~]', ' ', text)
    
    # remove extra whitespace in the middle of sentence
    text = re.sub('\s+', ' ', text)

    # remove whitespace in the beginning and end of sentence
    text = text.strip()
    
    return text

Text Normalization¶

Next, data preprocessing is carried out with a text normalization process to convert informal word forms into formal. Informal word forms that are often used by Indonesian netizens in writing tweets include:

  1. Words that show feelings (e.g. hiks (sad), wkwkwk/wkwk (laugh), huhuhu (cry), etc.);
  2. Repetition of characters to emphasize certain words (e.g. lamaaaa (long), sorryyyyy (sorry), etc.);
  3. Omission of vowels (e.g. gk (no), jgn (don't), etc.);
  4. Use of slang words (e.g. ciyus (serious), dah (already), etc.);
  5. Replacement of letters with numbers (e.g. s4ya (me), disk0n (discount), etc.).

We can use the typo and abbreviation dictionary obtained from the combined typo/abbreviation dictionary from Ibrohim and Budi (2019).

def normalize_text(text):
    return ' '.join(data_dict.get(word, word) for word in text.split())

The above function is written using Python list comprehension. If you are not familiar with it, you can refer to the following version:

def normalize_text(text):
    # temporary list
    result = []

    # split sentence into list of words
    split_text = text.split()

    # loop for each word
    for word in split_text:
        new_word = data_dict.get(word, word) # return normalized word if it exists in the dictionary
        result.append(new_word)

    # combine list of words into sentence
    combine_text = ' '.join(result)

    return combine_text

def remove_stopwords(text):

    result = []

    split_text = text.split()

    for word in split_text:

    combine_text = ' '.join(result)

    return combine_text

Tokenization & Word Embedding¶

Tokenization (tokenization) is a way of separating documents into smaller units called tokens. Generally, 1 token contains 1 word, but in some cases 1 token can contain 1 sentence.

After tokenization, the word embedding process can be done. Word embedding is the stage of representing a word in numerical form. Keep in mind that computers do not understand the meaning of words, so we need to convert each word into a numerical form. For classification cases, we can use the Bag of Words method.

Bag of Words¶

Bag of Words (BoW) counts the number of occurrences of a word in each document. Suppose we have two tweet documents as follows.

  1. sinyal kantor dan sinyal rumah sama sama jelek
  2. pakai wifi kantor tapi sinyal tetap jelek

Then BoW will create a table like the following.

sinyal kantor dan rumah sama jelek pakai wifi tapi tetap
Doc 0 2 1 1 1 2 1 0 0 0 0
Doc 1 1 1 0 0 0 1 1 1 1 1

We can create a Bag of Words using the CountVectorizer function from the scikit-learn library.

Count Unique Words¶

We can count the number of occurrences of a word from the whole document. Since each row is one document, and each column is one word, we can sum the contents of the values per column to get the total number of occurrences per word.

We can achieve this using the .sum() method. Parameters:

  • axis = 0 : sum all rows for each column
  • axis = 1 : sum all columns for each row

Wordcloud¶

From the number of word occurrences that we have obtained, we can visualize it in the form of wordcloud. Wordcloud forms a collection of words whose size indicates the frequency of occurrence. The larger the size of a word, the more frequently it appears. Vice versa, the smaller it is, the less frequent it is.

In [ ]:
from wordcloud import WordCloud

wordcloud = WordCloud().generate_from_frequencies(term_sum)
plt.imshow(wordcloud)
plt.axis("off")

(-0.5, 399.5, 199.5, -0.5)

We can change the background color of our wordcloud with the backgrund_color parameter.

In [ ]:
wordcloud = WordCloud(background_color='white').generate_from_frequencies(term_sum)
plt.imshow(wordcloud)
plt.axis("off")

(-0.5, 399.5, 199.5, -0.5)

To change the color of the words that appear, we can use the colormap parameter.

  • Color options: Choosing Colormaps in Matplotlib
  • Example of color in wordcloud: Wordcloud - colormap
In [ ]:
wordcloud = WordCloud(background_color='white', colormap='Blues').generate_from_frequencies(term_sum)
plt.imshow(wordcloud)
plt.axis("off")

(-0.5, 399.5, 199.5, -0.5)

Cross Validation¶

Cross validation is a resampling procedure used to evaluate machine learning models on limited data samples. The procedure involves dividing the dataset into two subsets.

  • Train data: Used to be learned by the model.
  • Test data: Used to evaluate the learned machine learning model.

The goal is to estimate the performance of the machine learning model on new data (data not used to train the model). Let's divide it using a proportion of 75:25

from sklearn.model_selection import train_test_split

Summary¶

This course provides a comprehensive grounding in data preparation, covering essential pre-processing techniques for both images and text. In the image preprocessing section, students learn about converting images to greyscale, understanding coordinate systems, applying geometric and morphological transformations, filtering images, and using thresholding methods. The text preprocessing section covers cleaning text with regular expressions, stemming to reduce words to their base forms, normalising text, removing common stop words, tokenising text into single words or terms, and constructing bag-of-words models for representing text. Together, these modules provide students with the skills needed to prepare raw unstructured data for advanced analysis, technique and modelling.
